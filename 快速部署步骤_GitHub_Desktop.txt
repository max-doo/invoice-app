  GitHub Desktop 快速部署步骤
═══════════════════════════════════════════════════════

📱 项目：在线发票开具应用
🌐 目标：部署到 GitHub Pages

═══════════════════════════════════════════════════════
  第一步：在 GitHub Desktop 中添加项目
═══════════════════════════════════════════════════════

1. 打开 GitHub Desktop
2. File → Add local repository
3. 选择文件夹：C:\Users\DMX\Desktop\invoice-app
4. 如提示创建仓库，点击 "create a repository"
5. 填写信息后点击 "Create Repository"

═══════════════════════════════════════════════════════
  第二步：提交代码
═══════════════════════════════════════════════════════

1. 在左下角 Summary 输入：配置 GitHub Pages 部署
2. 点击 "Commit to main" 按钮

═══════════════════════════════════════════════════════
  第三步：发布到 GitHub
═══════════════════════════════════════════════════════

1. 点击 "Publish repository" 按钮
2. 仓库名：invoice-app
3. 描述：在线发票开具应用
4. 决定是否勾选 "Keep this code private"
5. 点击 "Publish Repository"
6. 等待上传完成

═══════════════════════════════════════════════════════
  第四步：启用 GitHub Pages
═══════════════════════════════════════════════════════

1. Repository → View on GitHub (在浏览器打开)
2. 点击 Settings 标签
3. 左侧菜单点击 Pages
4. Source 选择：GitHub Actions
5. 自动保存

═══════════════════════════════════════════════════════
  第五步：触发部署
═══════════════════════════════════════════════════════

1. 点击 Actions 标签
2. 选择 "部署到 GitHub Pages" 工作流
3. 点击 "Run workflow" → "Run workflow"
4. 等待 1-3 分钟，看到绿色勾号

═══════════════════════════════════════════════════════
  第六步：访问网站 🎉
═══════════════════════════════════════════════════════

访问地址：
https://您的用户名.github.io/invoice-app/

例如用户名是 DMX：
https://DMX.github.io/invoice-app/

═══════════════════════════════════════════════════════
  后续更新（每次修改代码后）
═══════════════════════════════════════════════════════

1. 修改代码
2. 打开 GitHub Desktop
3. 输入提交信息
4. 点击 "Commit to main"
5. 点击 "Push origin" (顶部向上箭头按钮)
6. 等待 1-3 分钟自动部署完成

═══════════════════════════════════════════════════════
  常见问题速查
═══════════════════════════════════════════════════════

❌ 404 错误
   → 等待 5-10 分钟
   → 检查仓库名是否为 invoice-app
   → 检查 vite.config.ts 中 base 配置

❌ 样式丢失
   → 检查 vite.config.ts: base: '/invoice-app/'
   → 必须以 / 开头和结尾

❌ 认证失败
   → File → Options → Accounts
   → Sign out 后重新 Sign in

❌ 部署失败
   → 查看 Actions 错误日志
   → 本地测试: npm install && npm run build

═══════════════════════════════════════════════════════
  重要配置检查
═══════════════════════════════════════════════════════

✓ vite.config.ts 中的 base: '/invoice-app/'
✓ 仓库名必须与 base 中的名称一致
✓ GitHub Pages Source 设为 GitHub Actions
✓ 主分支名为 main（不是 master）
